# Stage 5: Polish and Deploy

## Current State
Core gameplay loop exists.

## Goal
Do small polish improvements and prepare simple static deployment.

## Tasks
1. Minor visual polish only (colors, spacing, labels).
2. Keep logic structure understandable for beginners.
3. Add short play instructions.
4. Deploy as static files (e.g., GitHub Pages).

## Acceptance Criteria
- Game is playable end-to-end.
- Student can explain core loop and key states.
- Project remains lightweight and dependency-free.

## Suggested Prompt for Codex
"Stage 5 polish only. Please make tiny readability/visual improvements without architecture changes, and provide a deployment checklist for static hosting."

## Common Mistakes
- Large refactor during final stage.
- Adding tooling not needed for static hosting.
- Polishing too much without validating gameplay.

## Recommended Git Checkpoint
`stage 5: polish and deploy`
